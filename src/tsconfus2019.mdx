## MakeCode Arcade with Static TypeScript

by Tom Ball ([@tballmsft](https://twitter.com/tballmsft)) and Holly Mitchell

- https://arcade.makecode.com/
- GUIでプログラミング(Blocks)してゲーム作る教育サービス
- BlocksとTypeScriptを相互変換、GUIに型アノテーションを活用
- 画面上はCanvasで動作するエミュレータを実装
- **Static TypeScript**でバイトコードにコンパイルして実デバイスで動く
- 画面で作成したゲームをWeb USB経由で実デバイスへインストール

## Testing Types: An Introduction to dtslint

by Dan Vanderkam ([@danvdk](https://twitter.com/danvdk?lang=en))

- 型パズルで生まれた**型定義のテスト**ってどうしてる？
- GenericsとConditional Typeで頑張って書く？
- 書けなくはないけど、エラーが理解しづらくて辛くない？
- 人間が読める型定義のテストを書くにはどうしたら良いか？そこで**dtslint**です
- DefinitelyTypedでも型定義のテストに使われている

---

## dtslint

- [microsoft/dtslint](https://github.com/microsoft/dtslint)

```ts
import { f } from "my-lib"; // f is(n: number) => void

// $ExpectType void
f(1);

// Can also write the assertion on the same line.
f(2); // $ExpectType void

// $ExpectError
f("one");
```
